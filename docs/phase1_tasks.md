# Phase 1: CLIプログラムでIPOを身につける (Day 1 - Day 5)

## 目的

**CLIのシンプルなプログラムを作りながら、IPO（Input-Process-Output）の考え方をしっかりと身につける。**

WebではなくCLI（黒い画面）でシンプルに学ぶことで、余計な要素に惑わされずにプログラミングの本質を理解します。

## 技術制限

- **言語:** Node.js（JavaScript）
- **環境:** コマンドライン（ターミナル）

**ステップ1: ファイルを作成**

```bash
# 出力:
```

```bash
# 1. 買い物に行く
# 2. 宿題をやる

node todo.js done 1
# 出力: タスク「買い物に行く」を完了しました

# 出力:
# === ToDoリスト ===
# 1. 宿題をやる
```

### 補足: データの揮発性について

このToDoリストには1つの特徴があります。プログラムを終了すると、追加したタスクが消えてしまいます。

```bash
node todo.js add 買い物
node todo.js list  # → 買い物が表示される
# ターミナルを閉じて再度開く
node todo.js list  # → 何も表示されない（消えている）
```

これは、データがメモリ（変数）に保存されているためです。Phase 2では、データを永続的に保存する方法（データベース）を学びます。

## Phase 1 まとめ: IPOの本質

Phase 1を完了したあなたは、以下を理解しました：

### IPOとは

```
Input（入力）  →  Process（処理）  →  Output（出力）
何を受け取る？     何をする？          何を返す？
```

### 学んだパターン

| ツール | Input | Process | Output |
|--------|-------|---------|--------|
| 挨拶ボット | 名前 | 条件分岐 | 挨拶文 |
| 割り勘計算機 | 金額・人数 | 計算・端数処理 | 金額表示 |
| ToDoリスト | コマンド・データ | 配列操作 | 結果表示 |

### IPOを使う理由

1. **設計がしやすい** - 作る前に「何を受け取り、何をして、何を返すか」を整理できる
2. **AIに指示しやすい** - IPOで伝えれば、AIが正確にコードを書いてくれる
3. **説明しやすい** - 他の人にコードの動きを説明できる

Phase 2では、**Webアプリの3層構造**を学びます。

```
Client（ブラウザ）  →  Server（Node.js）  →  Database（SQLite）
```

IPOの考え方は、Webアプリでも同じです。
ただし、「どこで」処理が行われるかを意識する必要があります。
