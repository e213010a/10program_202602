# 🏗️(工事中)　Phase 4: リノベーション・ハッカソン (Day 14)

## 目的

**Base Appを独自のプロダクトへ進化させる。**
これまで学んだIPO、3層構造、チーム開発の知識を総動員する。

---

## ハッカソン概要

### お題

> **「この無機質なリストアプリを、誰かのためのツールに作り変えよ」**

### ルール

| 項目 | 内容 |
|------|------|
| 制限時間 | **3時間** |
| チーム | 2〜3名 |
| スタート地点 | Base App（CRUD機能付きリスト） |
| 必須条件 | **1つ以上の機能追加（IPOの変更）** |

### 禁止事項

- 見た目（UI）の変更だけで終わらせること
- IPOカードなしでコードを書くこと
- mainに直接プッシュすること

---

| 時間 | 内容 | やること |
|------|------|---------|
| 0:00-0:30 | 企画 | アイデア出し、ターゲット決定 |

| 1:00-2:30 | 実装 | コーディング、PR、レビュー |
| 2:30-2:45 | 統合 | 全機能をmainにマージ |

## ステップ1: 企画（30分）

### 1-1. ターゲットを決める

「誰のための」ツールにするか？

**ターゲット例：**
- 忙しい大学生のための課題管理ツール
- 料理初心者のための買い物リスト
- ペットを飼っている人のためのお世話記録
- ゲーマーのためのプレイログ

### 1-2. アイデアシート

以下を埋めてください：

```
【ターゲット】
誰のためのツール？
→

【課題】
その人が困っていることは？
→

【解決策】
このツールでどう解決する？
→

【追加機能】
Base Appに何を追加する？
→
```

### 1-3. 追加機能のアイデア

**難易度: 低**
- カテゴリ分け（ラベル機能）
- 優先度の設定（高/中/低）
- 期限の設定
- 検索機能

**難易度: 中**
- 完了/未完了の切り替え
- ソート機能（日付順、優先度順）
- データのエクスポート
- 統計表示（完了率など）

**難易度: 高**
- 複数リストの管理
- 通知機能
- 共有機能

## ステップ3: 実装（90分）

### 3-1. ブランチ作成

各自が担当機能のブランチを作成：

```bash
git checkout main
git pull
git checkout -b feature/[機能名]
```

### 3-2. 実装のルール

1. **IPOカードに従って実装**
2. **主要な処理に日本語コメント**
3. **こまめにコミット**

### 3-3. コミットメッセージ例

```bash
git commit -m "feat: 優先度フィールドを追加

- Itemモデルにpriorityフィールドを追加
- 高/中/低の3段階で設定可能"
```

### 3-4. PR作成

```
タイトル: feat: 優先度機能を追加

## 変更内容
- アイテムに優先度（高/中/低）を設定できるように

## IPOカード
- Input: フォームで優先度を選択
- Process: サーバーがDBに優先度を含めて保存
- Output: リストに優先度バッジが表示される

## テスト方法
1. 新しいアイテムを追加（優先度:高を選択）
2. リストに赤いバッジで表示されることを確認
```

## ステップ4: 統合（15分）

### 4-1. マージ順序

1. 依存関係がない機能から順にマージ
2. コンフリクトがあればチームで解決
3. 全機能マージ後、動作確認

### 4-2. 最終チェック

```bash
git checkout main
git pull
npm start
# ブラウザで全機能の動作確認
```

## 評価基準

| 評価項目 | 配点 | 内容 |
|---------|------|------|
| **独自性** | 30点 | ターゲットと解決策が明確か |
| **機能追加** | 30点 | IPOの変更を伴う機能があるか |
| **チーム連携** | 20点 | Gitの履歴、PR、コンフリクト解決 |
| **コード品質** | 20点 | コメント、命名、構造 |

### 評価のポイント

**高評価:**
- ターゲットユーザーが明確
- 追加機能のIPOが説明できる
- チーム全員がコミットしている
- コードに日本語コメントがある

**減点対象:**
- 見た目だけの変更
- IPOカードがない
- mainへの直接プッシュ
- 一人だけが全部作っている
